Polygenic Risk Scores from GWAS Summary Statistics

SISG 2026 · Module QG3 · Summer Institute in Statistical Genetics

Methods: C+PT · SBLUP · SBayes C Tools: PLINK 1.9 · GCTB · R Validation: 1000 Genomes Project (N = 2,504) Repo: moeenriaz2/SISG26-PRS

Overview

This practical walks through three methods for computing polygenic risk scores (PRS) from GWAS summary statistics, evaluating each in a multi-ancestry validation cohort. No individual-level GWAS data is required — only published summary statistics and an LD reference panel.

MethodCore IdeaToolKey Parameter
C+PTSelect independent SNPs by LD clumping; tune P-value threshold in validationPLINK 1.9 + RP-value threshold T
SBLUPRidge regression on summary stats; approximates joint SNP effectsR (manual) or GCTAλ = M(1−h²)/(Nh²)
SBayes CBayesian two-component prior; estimates sparse joint effects via MCMCGCTBπ (causal fraction)
Learning Objectives
  • Understand how marginal GWAS effects differ from joint effects due to LD
  • Apply C+PT, SBLUP, and SBayes C to compute genome-wide PRS
  • Evaluate and compare PRS accuracy across five ancestry groups
  • Interpret the role of the LD reference panel and its limitations

Part 1: Clumping + P-value Thresholding (C+PT)

C+PT is the simplest and most widely used PRS method. It selects a set of independent SNPs associated with the trait at a given P-value threshold, then sums their marginal GWAS effects as the PRS weights.

Two-Step Procedure
C

Clumping (LD pruning)

Slide a window of ±500 kb. Within each window, keep the SNP with the smallest p-value; remove any SNP with r² > threshold (e.g., 0.1) relative to the index SNP. Ensures SNPs in ST are approximately independent.

T

Thresholding (P-value filter)

Try multiple P-value thresholds T ∈ {5×10⁻⁸, …, 0.5}. For each T, compute PRS in a validation sample and select the T that maximises R² (or AUC for binary traits).

Step 1.1 · Clumping with PLINK

We perform clumping using EUR individuals as the LD reference (matching the GWAS discovery ancestry).

bashTerminal
PLINK=./exe/plink
window_kb=1000   # 1000 kb = 1 Mb window around index SNP
r2_thresh=0.1    # LD threshold: remove SNPs with r² > 0.1
trait=Trait1
pv_thresh=5e-8   # start with genome-wide significance

${PLINK} --bfile 1kg_hm3 \
  --keep EUR.id \
  --clump ${trait}.fastGWA \
  --clump-kb  ${window_kb} \
  --clump-p1  ${pv_thresh} \
  --clump-p2  ${pv_thresh} \
  --clump-r2  ${r2_thresh} \
  --out C+PT/${trait}_rsq_${r2_thresh}_p_${pv_thresh}

# Output: C+PT/Trait1_rsq_0.1_p_5e-8.clumped
# Each row = one index SNP retained after clumping

Step 1.2 · Sweep across P-value thresholds

To find the optimal threshold, we loop over a range of P-value cutoffs for both traits:

bashTerminal
for pv_thresh in 5e-8 5e-7 5e-6 5e-5 5e-4 5e-3 5e-2 5e-1
do
  for trait in Trait1 Trait2
  do
    ${PLINK} --bfile 1kg_hm3 \
      --keep EUR.id \
      --clump ${trait}.fastGWA \
      --clump-kb  ${window_kb} \
      --clump-p1  ${pv_thresh} \
      --clump-p2  ${pv_thresh} \
      --clump-r2  ${r2_thresh} \
      --out C+PT/${trait}_rsq_${r2_thresh}_p_${pv_thresh} --silent
  done
done

Step 1.3 · Compute PRS with PLINK --score

Column specification for --score: col 2 = SNP ID, col 4 = effect allele A1, col 8 = BETA (marginal effect).

bashTerminal
for pv_thresh in 5e-8 5e-7 5e-6 5e-5 5e-4 5e-3 5e-2 5e-1
do
  for trait in Trait1 Trait2
  do
    clumped_file="C+PT/${trait}_rsq_${r2_thresh}_p_${pv_thresh}.clumped"
    # Skip if no SNPs were retained at this threshold
    [ ! -f "${clumped_file}" ] && continue

    ${PLINK} --bfile 1kg_hm3 \
      --score ${trait}.fastGWA 2 4 8 sum center \
      --extract ${clumped_file} \
      --out C+PT/${trait}_rsq_${r2_thresh}_p_${pv_thresh}.pred --silent
  done
done
# Output: *.pred.profile  →  column SCORESUM = individual PRS

Step 1.4 · Evaluate prediction accuracy in R

Load all PRS profiles and compute R² within each ancestry group.

RR
options(stringsAsFactors = FALSE)
library(data.table)

r2_thr  <- 0.1
trait   <- "Trait2"          # switch to "Trait1" to compare
threshs <- c("5e-8","5e-7","5e-6","5e-5","5e-4","5e-3","5e-2","5e-1")

# Ancestry labels
pops <- fread("1kg-sample-2504-phased.txt")

# Phenotype
phen <- fread(paste0("1kg.", trait, ".phen"))
phen <- phen[, .(IID = V2, PHENO = V3)]

# Merge PGS from all thresholds
for (pv in threshs) {
  f <- paste0("C+PT/", trait, "_rsq_", r2_thr, "_p_", pv, ".pred.profile")
  if (!file.exists(f)) next
  tmp <- fread(f)[, .(IID = V2, PGS = SCORESUM)]
  setnames(tmp, "PGS", paste0("P_", pv))
  phen <- merge(phen, tmp, by = "IID", all.x = TRUE)
}

# Attach ancestry
phen <- merge(phen, pops[, .(IID = sample, super_pop)], by = "IID")

# ---------- Compute R² per threshold per ancestry ----------
ancestries <- c("EUR", "SAS", "EAS", "AMR", "AFR")
pgs_cols   <- grep("^P_", names(phen), value = TRUE)

r2_table <- sapply(ancestries, function(anc) {
  sub <- phen[super_pop == anc]
  sapply(pgs_cols, function(col) {
    if (all(is.na(sub[[col]]))) return(NA)
    cor(sub$PHENO, sub[[col]], use = "complete.obs")^2
  })
})
rownames(r2_table) <- pgs_cols
print(round(r2_table, 4))

# Best threshold overall
best <- pgs_cols[which.max(rowMeans(r2_table, na.rm = TRUE))]
cat("\nBest threshold:", best, "\n")

Part 2: Summary-data BLUP (SBLUP)

SBLUP is a ridge-regression approximation of the individual-level BLUP (Best Linear Unbiased Prediction). It uses GWAS summary statistics together with an LD correlation matrix to compute joint SNP effects — accounting for LD between SNPs rather than using marginal effects directly.

Shrinkage Parameter λ
SymbolMeaningThis Practical
MNumber of SNPs32,260 (chr20 HapMap3)
h²SNP heritability0.2 (both traits)
NGWAS sample size348,501
λRidge penalty = M(1−h²)/(Nh²)≈ 0.463

Step 2.1 · (Reference) Computing the LD matrix with PLINK

In this practical we use a pre-computed LD matrix. The command below shows how it was created (⚠ takes ~30 min; do not run during the session):

bashTerminal — for reference only
# Computed on N=348,501 UKB EUR individuals — provided pre-computed
${PLINK} \
  --bfile UKBu_chrom20 \
  --chr 20 \
  --ld-window 999999999 \
  --ld-window-kb 1000 \
  --ld-window-r2 0.00 \
  --r 'yes-really' 'gz' \
  --out plinkLDMat_chrom20
# Output: plinkLDMat_chrom20.ld.gz
# Columns: CHR_A BP_A SNP_A CHR_B BP_B SNP_B R
⚠ Using 1KG as LD Reference (Practical Alternative)

If you don't have the pre-computed UKB LD matrix, you can compute LD from the 1KG EUR samples as an approximation (N = 503). This will produce noisier estimates due to the smaller reference sample:

${PLINK} --bfile 1kg_hm3 --keep EUR.id \
  --chr 20 --ld-window 999999999 --ld-window-kb 1000 \
  --ld-window-r2 0.00 --r 'yes-really' 'gz' --out plinkLDMat_chrom20

Step 2.2 · Compute SBLUP weights in R

The SBLUP computation is handled by the provided R script. Run it for each trait:

bashTerminal
for trait in Trait1 Trait2; do
  Rscript scripts/sblup.R \
    --trait ${trait} \
    --gwas  ${trait}.fastGWA \
    --ldmat plinkLDMat_chrom20.ld.gz \
    --h2    0.2 \
    --N     348501 \
    --outdir SBLUP/
done
# Output: SBLUP/Trait1.sblup.weights  (columns: SNP A1 BETA_SBLUP)

The script scripts/sblup.R is included in the repository. The core computation is:

RR (sblup.R — key lines)
## SBLUP core computation
M      <- nrow(gwas)            # number of SNPs
lambda <- M * (1 - h2) / (N * h2)   # shrinkage parameter

## Build LD matrix R from PLINK --r output
R_mat  <- build_ld_matrix(ld_raw, snp_ids = gwas$SNP)

## Solve: beta_joint = (R + lambda * I)^{-1} * beta_marginal
A_mat        <- R_mat + lambda * diag(M)
beta_sblup   <- solve(A_mat, gwas$BETA)

## Write PLINK --score compatible output
write.table(
  data.frame(SNP = gwas$SNP, A1 = gwas$A1, BETA = beta_sblup),
  output_file, quote = FALSE, row.names = FALSE
)

Step 2.3 · Compute SBLUP PRS with PLINK

bashTerminal
for trait in Trait1 Trait2; do
  ${PLINK} --bfile 1kg_hm3 \
    --score SBLUP/${trait}.sblup.weights 1 2 3 sum center \
    --out   SBLUP/${trait}.pred
done
# Output: SBLUP/Trait1.pred.profile  →  SCORESUM column = SBLUP PRS

Alternative: SBLUP via GCTA (optional)

GCTA implements SBLUP via its --cojo-sblup flag using individual-level data directly (λ = M(1/h² − 1)):

bashTerminal — optional, ~5 min
GCTA=./exe/gcta-1.94.1
lambda=135628    # M * (1/h2 - 1) = 32260 * 4.0 ≈ 129040

for trait in Trait1 Trait2; do
  ${GCTA} \
    --bfile 1kg_hm3 \
    --chr 20 \
    --cojo-file ${trait}.ma \
    --cojo-sblup ${lambda} \
    --cojo-wind 1000 \
    --thread-num 4 \
    --out SBLUP/${trait}_gcta
done

Part 3: SBayes C (GCTB)

SBayes C is a Bayesian approximation of the individual-level Bayes C model. Unlike SBLUP's infinitesimal prior (all SNPs causal), SBayesC assumes only a proportion π of SNPs affect the trait. This is more realistic for most complex traits.

MCMC Parameters Explained
FlagMeaningValue Used
--chain-lengthTotal MCMC iterations10,000
--burn-inIterations discarded as warm-up5,000
--thinKeep every nth sample10
--piStarting value for causal fraction0.0001
--hsqStarting value for h²0.001
--robustRobust mode for numerical stability—

Step 3.1 · Run SBayes C with GCTB

bashTerminal
GCTB=./exe/gctb
mldm=ldm/mldm.txt     # list of paths to 38 LD block matrices

for trait in Trait1 Trait2; do
  ${GCTB} \
    --sbayes C \
    --mldm   ${mldm} \
    --gwas-summary ${trait}.ma \
    --pi     0.0001 \
    --hsq    0.001 \
    --chain-length 10000 \
    --burn-in      5000  \
    --no-mcmc-bin \
    --robust \
    --out-freq 100 \
    --thin     10  \
    --out SBC/${trait}
done

# Key output files:
#   SBC/Trait1.parRes     → posterior estimates of h², π, σ²_b, etc.
#   SBC/Trait1.snpRes     → posterior mean effect per SNP
#   SBC/Trait1.mcmcsamples → MCMC trace (for convergence checks)

Method Comparison

The code below builds a combined accuracy table for all three methods and both traits:

RR
library(data.table); library(ggplot2)

pops <- fread("1kg-sample-2504-phased.txt")
ancestries <- c("EUR","SAS","EAS","AMR","AFR")

results <- list()
for (trait in c("Trait1","Trait2")) {
  phen <- fread(paste0("1kg.", trait, ".phen"))[, .(IID=V2, PHENO=V3)]

  ## Best C+PT
  best_cpt_file <- # (replace with best threshold from Part 1)
    paste0("C+PT/", trait, "_rsq_0.1_p_5e-4.pred.profile")
  cpt  <- fread(best_cpt_file)[, .(IID=V2, CPT=SCORESUM)]

  ## SBLUP
  sblup <- fread(paste0("SBLUP/", trait, ".pred.profile"))[, .(IID=V2, SBLUP=SCORESUM)]

  ## SBayes C
  sbc   <- fread(paste0("SBC/",  trait, ".pred.profile"))[, .(IID=V2, SBC=SCORESUM)]

  dat <- phen |> merge(cpt, by="IID") |> merge(sblup, by="IID") |>
                 merge(sbc, by="IID") |>
                 merge(pops[, .(IID=sample, super_pop)], by="IID")

  for (anc in ancestries) {
    s <- dat[super_pop == anc]
    results[[length(results)+1]] <- data.table(
      Trait = trait, Ancestry = anc,
      CPT   = cor(s$PHENO, s$CPT)^2,
      SBLUP = cor(s$PHENO, s$SBLUP)^2,
      SBC   = cor(s$PHENO, s$SBC)^2
    )
  }
}
tab <- rbindlist(results)
print(tab)

## Plot
tab_long <- melt(tab, id.vars=c("Trait","Ancestry"),
                 variable.name="Method", value.name="R2")
ggplot(tab_long, aes(x=Ancestry, y=R2, fill=Method)) +
  geom_bar(stat="identity", position="dodge") +
  facet_wrap(~Trait) +
  scale_fill_manual(values=c(CPT="#1D4E89", SBLUP="#0A6640", SBC="#6D28D9")) +
  labs(title="PRS Prediction Accuracy by Method and Ancestry",
       y="R² (prediction accuracy)", x="Super-population") +
  theme_minimal(base_size=13) +
  theme(legend.position="top")
C+PT
  • ✅ Simple, no model assumptions
  • ✅ Fast, works with any GWAS format
  • ✅ Interpretable (clear SNP selection)
  • ⚠ Requires validation sample
  • ❌ Discards LD-correlated SNPs → information loss
  • ❌ Marginal (not joint) effects → noisy in polygenic traits
SBLUP
  • ✅ Uses all SNPs jointly (optimal linear)
  • ✅ No validation sample needed
  • ✅ Fast closed-form solution
  • ⚠ Requires large, matched LD reference
  • ❌ Infinitesimal prior — suboptimal for sparse traits
  • ❌ Needs known h²
SBayes C
  • ✅ Most flexible prior (estimates π from data)
  • ✅ Better accuracy for sparse-to-moderate architecture
  • ✅ Estimates h², π, Vg as a byproduct
  • ⚠ Requires GCTB LD block matrices
  • ❌ Slower (MCMC sampling)
  • ❌ Convergence can fail with bad LD reference
